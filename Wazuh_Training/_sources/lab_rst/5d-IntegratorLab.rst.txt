The teacher will share the invite link to the Slack Workspace where we will configure our environments to send alerts to.

We will be using the #wazuh-alert channel for our Slack lab.

You may create your own Slack App with an Incoming Webhook so you may configure your environment to send alerts by using the following configuration on your manager's :file:`ossec.conf`

.. code::

	<!-- Slack ssh basic fail to green channel -->
	<integration>
	  <name>slack</name>
	  <hook_url>YOUR_WEBHOOK</hook_url>
	  <rule_id>5710</rule_id>
	  <alert_format>json</alert_format>
	</integration>


Restart the manager and watch the Slack channel for alerts to start appearing every time one of our linux systems has a failed ssh login attempt involving a nonexistent user name.


**Virus Total**


You may sign up for a free VirusTotal account and add the following configuration to your environment:

.. code::

    <ossec_config>
      <integration>
        <name>virustotal</name>
        <api_key>${your_virustotal_api_key}</api_key>
        <group>syscheck</group>
        <alert_format>json</alert_format>
      </integration>
    </ossec_config>

Then create files in a folder monitored in realtime or with whodata (such as `C:\apple` or `/apple` from the previous exercises) and see how Wazuh handles the alert information and provides context from the VirusTotal API response.

On the Linux agent you may create a positive detection by using the EICAR test file:

.. code::

	curl -o /apple/worm https://secure.eicar.org/eicar.com

Note that the free VirusTotal account is limited to 4 requests per minute.
